## 1. 系统架构概述

### 1.1 核心业务流程

```
创建学员 → 创建排班 → 学员预约(排队) → 满员截止 → 签到 → 计时训练 → 签退 → 学时累计 → 告警检查
     ↓           ↓          ↓
   学员档案    教练排班    先到先得
```

### 1.2 数据库核心表

| 表名 | 说明 | 关键字段 |
|------|------|---------|
| users | 用户表(管理员/教练) | id, username, role, name |
| students | 学员表 | id, name, training_type, stage, coach_id |
| schedules | 排班表 | id, coach_id, course_type, schedule_date, start_time, end_time, max_students, current_students, status |
| bookings | 预约表 | id, schedule_id, student_id, status |
| attendance | 出勤表 | id, student_id, schedule_id, coach_id, course_type, check_in_time, check_out_time, duration_hours, status |
| alert_rules | 告警规则 | course_type, required_hours, warning_threshold |
| alerts | 告警记录 | student_id, course_type, current_hours, required_hours, severity, status |

| 优先级 | P1 |

**告警严重级别计算逻辑：**
```
ratio = current_hours / required_hours
ratio < 0.5       → severity = high (严重不足)
0.5 ≤ ratio < 0.8 → severity = medium (中度不足)
ratio ≥ 0.8       → severity = low (接近目标)
已达 required_hours → status = resolved
```
